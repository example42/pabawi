Impact Analysis

Puppet Exit Code Normalization

The normalizePuppetExitCodes() function is called in runPuppetOn() which is the single entry point for all Puppet agent runs. It mutates the ExecutionResult in-place before persisting to the database. This affects:

  • Execution history records (status field)
  • Journal entries (success vs failed)
  • SSE streaming events (complete event status)

Blast radius: moderate — changes how Puppet run results are classified system-wide.

ANSI Rendering

Uses ansiToHtml() from frontend/src/lib/ansiToHtml.ts with {@html} rendering. The function must sanitize HTML entities before inserting ANSI spans to prevent XSS. Four components affected — all output display paths.

Blast radius: contained — purely visual, no data flow changes.

Filter Layout Refactor

Pure layout restructuring in HomePage and InventoryPage. Source filter badges moved from inline with other controls to a dedicated second row. No logic changes — same state variables, same filtering behavior.

Blast radius: minimal — visual only.

Proxmox Snapshot Action

Adds "snapshot" to the Zod enum in the route validation schema. The backend handler dispatches to ProxmoxService which must already support this action for it to work end-to-end.

Blast radius: minimal — additive schema change.
