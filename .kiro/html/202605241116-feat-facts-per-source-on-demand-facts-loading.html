Commit Message

feat(facts): per-source on-demand facts loading

The Facts tab on the node detail page no longer fans out to every
information source on page load. Each integration now exposes its
own card with a Load / Refresh / Retry control, and the existing
"Gather Facts" button is repurposed as "Load all".

Backend:
- GET /api/nodes/:id/facts now accepts ?source=<name> to scope a
  request to a single information source. The default fan-out skips
  bolt/ssh/ansible (active sources that establish a remote
  connection) so navigation no longer triggers SSH sessions. Returns
  404 UNKNOWN_SOURCE when the named source is not registered.
- ProxmoxIntegration.getNodeFacts() now resolves non-canonical
  identifiers (hostname, linked node name) via the Proxmox inventory
  before forwarding to the service, mirroring AWS/Azure. Unknown IDs
  throw "Proxmox node not found: <id>" and surface under
  errors.proxmox instead of the previous parse-error crash.

Frontend:
- NodeDetailPage replaces the single bulk-load flow with per-source
  loading state. availableFactSources is derived from the
  integrations endpoint; loadFactsForSource() targets ?source=name;
  loadAllFactSources() optionally triggers the active gather pipeline
  before fanning out per-source GETs.
- MultiSourceFactsViewer reworked: per-source cards, click-to-focus
  (full-card role="button" target), retry/refresh affordances,
  third "All" toggle that merges fact keys across sources by stable
  JSON equality.

Tests / infra:
- New backend/test/routes/facts-source-filter.test.ts pins the
  ?source= contract (skip-active default, 404 UNKNOWN_SOURCE,
  per-source scoping).
- Two new ProxmoxIntegration test cases cover inventory-resolved
  lookup and the not-found error path.
- AuthenticationService accepts an explicit bcryptCostFactor and
  defaults to cost 4 under NODE_ENV=test (cost 12 in production).
  Hashing dominated test runtime; the password-hashing property test
  pins the production contract by constructing a service with cost
  12 explicitly.
- backend/vitest.config.ts caps maxWorkers and switches to forks to
  reduce CPU contention and ephemeral-port exhaustion under
  supertest.
- 404 latency assertion in api-performance.test.ts relaxed from
  100ms to 250ms to absorb host-load variance.
- group-expansion property arbitraries deduplicated by group id to
  avoid invalid fixtures with duplicate group ids.
- E2E inventory-to-facts spec rewritten for the new flow.
- CHANGELOG entries added under 1.3.0; release date moved to
  2026-05-25.

Pages Affected

  • Primary: http://localhost:5173/nodes/:id — Node detail page, Facts tab. Direct rewrite of the facts loading flow (NodeDetailPage.svelte, MultiSourceFactsViewer.svelte, routes/facts.ts).
  • Indirect: http://localhost:5173/inventory — Proxmox cross-source identifier resolution affects the path from inventory entries (where Proxmox guests can show up under a hostname/linked-node-name) into the facts view.
  • Cross-cutting: http://localhost:5173/login — AuthenticationService constructor signature gained an optional bcryptCostFactor. Production behaviour is unchanged (cost 12 default outside NODE_ENV=test); login latency and hash format are identical.

What to Check

  • Node detail / Facts tab — open a node, switch to Facts. Confirm:
    • No automatic loading on tab open. Each source card sits idle with a "Load facts" button.
    • Per-source Load works individually for Bolt, SSH, Ansible (active sources) and PuppetDB / Puppetserver / Hiera / Proxmox / AWS / Azure (passive sources).
    • "Load all" button triggers the gather pipeline plus per-source fan-out.
    • Clicking anywhere on a loaded source card focuses that source (active highlight moves).
    • "Per Source · All · Merged" toggle: All view enumerates every fact key across sources; identical values collapse into one row with a badge per contributing source.
    • Per-card Refresh and Retry on errored sources.
    • Page load no longer establishes any SSH connection (verify on a node configured for SSH).
  • Proxmox-backed nodes — open a node whose ID is a hostname (not the canonical proxmox:{node}:{vmid} form). Click "Load facts" on the Proxmox card. Verify facts load instead of crashing with a parse error. Open a node with a hostname that no Proxmox guest matches and verify a clean per-card error: Proxmox node not found: ….
  • Login flow — sign in with an existing user. bcrypt cost 12 still applies in production builds; latency should match prior runs (check server logs for any "Invalid token signature" or hash-mismatch errors).
  • Test suite — npm test should pass on developer machines noticeably faster (forks pool + bcrypt cost 4 in tests). The 404 perf assertion now permits up to 250 ms.

Impact Analysis

Blast radius — moderate, scoped to the Facts feature plus a small infra knob.

  • API contract change (additive): GET /api/nodes/:id/facts default behaviour now skips bolt/ssh/ansible. Any caller that relied on the old default fan-out for active sources must add ?source= or call POST /api/nodes/:id/facts first. Frontend already updated; no other internal callers inspected this endpoint with implicit-active expectations. The contract is pinned by backend/test/routes/facts-source-filter.test.ts.
  • Plugin behaviour change: only ProxmoxIntegration.getNodeFacts changed. AWS/Azure already had this pattern; SSH/Bolt/Ansible/Puppet sources are untouched. The new code path adds a single inventory call (cached in the service) before hitting the API.
  • Auth service signature change: the constructor gained a fourth optional parameter. All existing callers (DI container, tests) continue to work unchanged. The production cost factor is unchanged at 12; test runs use 4 only when NODE_ENV=test. The bcrypt format / hash compatibility is preserved.
  • Frontend component: MultiSourceFactsViewer props changed (availableSources, loadingStates, onLoadSource, onLoadAll, loadingAll replace loading, onGatherFacts, gatheringFacts). The only consumer is NodeDetailPage.svelte, also updated in this commit. No external consumers exist.
  • Vitest config: changes test-runner concurrency only. No production impact. Worker cap is min(8, ceil(cpus / 2)), fork pool, min 1 worker — protects CI and developer machines from supertest port exhaustion and bcrypt-driven CPU thrash.
