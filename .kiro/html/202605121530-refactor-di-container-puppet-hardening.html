Impact Analysis

This is a cross-cutting infrastructure change. The DI container touches every route factory in the backend (27 files), but the change is mechanical: each factory gains an optional container parameter with a default fallback. Runtime behavior is preserved — services are the same instances, just resolved from the container instead of instantiated inline.

The puppet route rewrite is more substantial: it changes the function signature from (boltService, ...) to (integrationManager, ...), adds multi-node support, and introduces Zod validation schemas. This is isolated to the puppet execution path.

The frontend changes are component-level extractions and prop additions — no routing or state management changes.

Blast radius: High file count but low behavioral risk. The DI container provides backward-compatible defaults. The puppet route is the only path with new logic.
