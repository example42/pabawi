Impact Analysis

Backend: puppet.ts refactor

The runPuppetOn helper is a pure extraction — no behavioral change. It's called from within createPuppetRouter, which is consumed only by startServer. Blast radius is contained to the puppet execution path. The interface is exported for future testability but has no external consumers yet.

Frontend: API module split

This is a cross-cutting change affecting 15+ frontend files. All provisioning-related functions moved from api.ts to proxmoxApi.ts, awsApi.ts, and azureApi.ts. The HTTP infrastructure (get, post, put, del) remains in api.ts and is imported by the new modules. Any component that previously imported provisioning functions from api.ts must now import from the correct domain module.

Frontend: localStorage hardening

Isolated to expertMode.svelte.ts and logger.svelte.ts. These are singletons used across all pages, but the change is defensive (adds try/catch around localStorage access). No behavioral change in normal browser environments.

Test infrastructure

The new setup.ts provides a working localStorage mock for all frontend tests via vitest setupFiles. This eliminates a class of test failures in Node.js 20+ where the built-in localStorage global has non-functional methods.
